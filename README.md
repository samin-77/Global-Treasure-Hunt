
🚀 Project Reveal: Building a Browser-Based Global Treasure Hunt! 🌍🏴‍☠️

I’m excited to share a project I’ve been working on: The Treasure Hunt Map Game. It’s an interactive, geography-based puzzle game where players travel the virtual world to uncover hidden riches by solving clues about iconic landmarks.

🕹️ How to Play:
Explore the Map: Navigate a global map to find hidden treasure chests.

Unlock the Clue: Click a chest to reveal a riddle about a famous world location.

Solve & Earn: Type the correct answer to claim the treasure, earn points, and trigger a celebration!

Persistent Progress: Your score and discovered treasures are saved automatically, so you can pick up where you left off.

🛠️ Tech Stack:
To keep the project lightweight and lightning-fast, I built this entirely on the frontend:

Vanilla JavaScript: Managed the game logic, state handling, and answer validation.

Leaflet.js / OpenStreetMap API: Integrated for smooth, interactive mapping and custom marker placement.

Tailwind CSS: Used for a modern, responsive UI and a "treasure-map" themed design.

Browser LocalStorage: Implemented for persistent data storage without needing a backend database.

Canvas Confetti: Added for that extra "win" feeling!

🧠 The "How" & The Challenges:
The goal was to create a "serverless" experience that works immediately in any browser.

The biggest challenge? Managing the game state across different screen sizes while ensuring the map remained intuitive. I had to implement a custom coordinate system for markers and ensure the UI was fully responsive—switching from a wide-screen dashboard to a mobile-friendly "Discovery Log." I also focused on Answer Normalization, ensuring the game is smart enough to accept "Eiffel Tower" regardless of case sensitivity or accidental spaces.

This project was a fantastic exercise in integrating third-party APIs and managing local data persistence. It's amazing what you can build with pure frontend technologies!
